# Headline Skill 介绍

`headline-skill` 不是帮你临时起几个标题的 prompt，而是一套可迁移的标题工作流。

## 它解决什么问题

很多 AI 写出来的标题不差，但就是不想点。问题通常不在语法，而在表达太老实。它太快把中心思想讲完，于是标题变成了摘要、结论，或者教程目录。

这个项目处理的不是某一种固定风格，而是标题生产这件事本身。它把稳定的部分沉到流程里，把会因平台、受众和业务变化的部分放到 profile 里，把能量化的判断放进 scorer。这样一来，系统可以复用，审美可以替换。

## 它怎么工作

整个仓库分三层。

第一层是流程。负责读 brief、拆卖点、选 hook route、生成候选、做硬过滤、做软评审，然后给出首选和备选。

第二层是 profile。这里放平台规则、受众画像、语气边界、禁词、样本和评分权重。换个平台，换的是这一层，不是整条 workflow。

第三层是 scorer。像节奏、禁词、重复度这类本来容易靠感觉判断的东西，尽量交给脚本处理，减少标题系统越来越像拍脑袋的风险。

## 仓库里有什么

- `SKILL.md`
  - Codex Skill 主入口，只负责调用规则和工作流程
- `profiles/`
  - 放不同平台和受众的配置
- `references/`
  - 放通用方法、profile schema 和 review rubric
- `scripts/`
  - 放硬过滤、节奏评分、结果合并和校验脚本
- `tests/`
  - 放 golden set 和单元测试

## 适合谁用

这个项目最适合两类人。

第一类是做内容的人，想把起标题从经验活变成流程活。第二类是想把自己的标题方法开源出去的人，想让别人接入自己的风格，而不是只能复刻自己的口味。

## 为什么适合开源

很多标题方案一旦离开原作者就没法用，因为方法、审美和规则全塞在一个 prompt 里。别人想接入，只能整套拆掉重写。

这个项目反过来设计。流程固定，profile 可换，scorer 可插。别人接入时主要改自己的场景、样本和边界，不需要重做底层逻辑。这才是它的开源价值。
