# Headline Skill 介绍

`headline_skill` 想做的事很直接：把起标题从一次性的提示词技巧，做成一套可复用的工作流。

它不预设你的审美，也不试图替你定义什么叫好标题。它做的是把底层流程固定下来，把真正会因人而异的部分单独抽出来，让你和别人都能按自己的业务接进去。

这套设计分三层。第一层是流程，负责读 brief、拆信息、挑路线、生成候选、做过滤和选优。第二层是 profile，负责平台规则、受众画像、语气限制、禁词、样本和权重。第三层是 scorer，负责把一些本来靠感觉判断的东西，尽量变成脚本和数字。

这个项目最适合两类人。一类是自己做内容，想把起标题这件事系统化；另一类是想把自己的标题方法开源出去，但又不想把别人绑死在自己的风格上。

你可以把它理解成一个标题框架，而不是一个标题模板。流程可以复用，风格可以替换，检查可以扩展。这样项目才有开源价值。
